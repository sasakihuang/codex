# 05. 基础配置: General、个性化、AGENTS.md

Codex 的配置分三层：当前对话里的要求、仓库里的 AGENTS.md、全局/项目设置。新手最应该先学 AGENTS.md。

## 三层配置

| 层级 | 适合放什么 | 举例 |
| --- | --- | --- |
| 当前提示词 | 这次任务临时要求 | “只改 docs 目录” |
| AGENTS.md | 仓库长期约定 | 测试命令、代码风格、安全边界 |
| 全局设置 | 个人默认偏好 | 默认语言、权限习惯、模型偏好 |

## AGENTS.md 是什么

AGENTS.md 可以理解为“写给 AI 代理看的 README”。它告诉 Codex：

- 项目结构。
- 常用命令。
- 代码风格。
- 哪些文件不能碰。
- 怎么验证完成。
- 输出格式偏好。

## 推荐 AGENTS.md 模板

下面以一位大学辅导员用 Codex 处理带班事务为例（换成你自己的项目同理）：

```md
# AGENTS.md

## Project

这是一位大学辅导员的「带班工作」项目：用 Codex 整理学生名单、统计分组、
汇总通知材料，并把学期事务生成可直接报送的总结报告。
资料以 Excel、Word、PDF 和 Markdown 为主，不需要任何编程基础。

## Commands

- 本地预览：python3 -m http.server 8765
- 统计分组：让 Codex「把『学生名单.xlsx』按专业和宿舍分组并统计人数」
- 生成材料：让 Codex「根据本月台账写一份 800 字主题班会总结」

## Rules

- 默认中文，语气正式、可直接用于学院报送材料。
- 不要改动「原始台账」目录，所有结果都放到「整理后」目录。
- 学号、身份证号、电话、成绩、家庭情况属于隐私，不写进公开文件或网页。
- 每次改完，说明改了哪些文件、数据来自哪张表、怎么核对。

## Done

- 报表和总结格式统一、数据可核对，能直接报送。
- 名单、台账、通知分类清晰，方便检索。
- 没有泄露任何学生隐私信息。
```

## 个性化设置怎么写

适合放：

- 默认用中文解释。
- 回答要简洁。
- 代码改动后列文件。
- 调试时给假设、实验和最小修复。

不适合放：

- 密钥。
- 私人账号密码。
- 一次性任务要求。
- 已经过期的下载地址。
